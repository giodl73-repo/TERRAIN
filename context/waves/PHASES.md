# TERRAIN Phases

Work is organized into American expansion/infrastructure-history themed phases
and small pulses. The names are planning mnemonics for survey, exploration,
movement, ownership, and connection work; they are not endorsements of the
historical harms or politics of expansion.

## Protocol

1. Read this file.
2. Read the active phase `WAVE.md`.
3. Read the target pulse under `pulses/`.
4. Implement the smallest complete slice.
5. Keep product-specific behavior out of shared kernels.
6. Update docs and wave/pulse status.
7. Run the repo validation commands.
8. Commit when green.
