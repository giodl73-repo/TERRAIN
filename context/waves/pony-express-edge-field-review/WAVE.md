# Pony Express - Edge Field Review

## Goal

Turn edge audit findings into field-manager actions that can be discussed in an
operating meeting without graph vocabulary.

## Why now

Cumberland Gap made cut edges and disconnected territories visible. The next
step is communication: managers need to know whether a finding is acceptable,
risky, or caused by bad input before they decide to reassign sites.

## Pulses

| Pulse | Title | Status | Outcome |
|---:|---|---|---|
| 01 | Edge field review recommendations | done | Translated audit findings into recommendation and action language. |
| 02 | Edge field review packet | done | Packaged field-review text, action rows, audit rows, and edge visuals. |
| 03 | Role review and docs | done | Documented manager-language semantics and review boundaries. |

## Acceptance

- Edge field review explains cut edges, disconnected territories, and unknown
  edge sites in field-manager language.
- Packet output includes field-review text, action rows, audit rows, and edge
  visual artifacts.
- Manager wording stays in TERRAIN; METIS-CORE remains a graph/partition
  dependency only.

## Non-goals

- TERRAIN does not turn cut edges into automatic reassignment rules.
- TERRAIN does not move manager review wording into shared graph kernels.
- TERRAIN does not add turn-by-turn routing.
